## 4. 도메인 모델

### 4.1 User

| 속성명 | 타입 | 제약 | 설명 |
|--------|------|------|------|
| user_id | UUID | PK, Not Null | 사용자 고유 식별자 |
| email | String | Unique, Not Null | 로그인 ID로 사용되는 이메일 주소 |
| password_hash | String | Not Null | bcrypt 해시 처리된 비밀번호 |
| name | String | Not Null, max 50자 | 사용자 표시 이름 |
| status | Enum | Not Null, Default: active | 계정 상태 (active \| withdrawn) |
| created_at | DateTime | Not Null | 계정 생성 일시 |
| updated_at | DateTime | Not Null | 최종 수정 일시 |
| withdrawn_at | DateTime | Nullable | 탈퇴 처리 일시 |

### 4.2 Category

| 속성명 | 타입 | 제약 | 설명 |
|--------|------|------|------|
| category_id | UUID | PK, Not Null | 카테고리 고유 식별자 |
| user_id | UUID | FK → User.user_id, Not Null | 카테고리 소유 사용자 |
| name | String | Not Null, max 50자, Unique within user | 카테고리 이름 |
| is_default | Boolean | Not Null | 기본 카테고리 여부 (true: 삭제·수정 불가) |
| created_at | DateTime | Not Null | 카테고리 생성 일시 |

### 4.3 Todo

| 속성명 | 타입 | 제약 | 설명 |
|--------|------|------|------|
| todo_id | UUID | PK, Not Null | 할일 고유 식별자 |
| user_id | UUID | FK → User.user_id, Not Null | 할일 소유 사용자 |
| category_id | UUID | FK → Category.category_id, Not Null | 연결된 카테고리 |
| title | String | Not Null, max 200자 | 할일 제목 |
| description | String | Nullable | 할일 상세 설명 |
| due_date | Date | Not Null | 종료 예정일 (KST 기준 오늘 이상) |
| is_completed | Boolean | Not Null, Default: false | 완료 여부 |
| created_at | DateTime | Not Null | 할일 생성 일시 |
| updated_at | DateTime | Not Null | 최종 수정 일시 |

## 5. 엔티티 관계

```
User (1) ──────────── (N) Category
 │                         │
 │                         │ (category_id FK)
 │                         │
 └──────── (N) Todo ───────┘
              (user_id FK)
```

- User : Category = 1 : N (한 사용자는 여러 카테고리를 가짐)
- User : Todo = 1 : N (한 사용자는 여러 할일을 가짐)
- Category : Todo = 1 : N (한 카테고리에 여러 할일이 속함)
- Todo는 반드시 하나의 Category에 속해야 함 (category_id Not Null)

## 6. 핵심 비즈니스 규칙

| BR-ID | 규칙 | 관련 UC-ID |
|-------|------|------------|
| BR-01 | 모든 기능은 인증된(status=active) 사용자만 사용 가능 | UC-03~UC-12 |
| BR-02 | 사용자는 자신 소유(user_id 일치) 리소스만 접근 가능 | UC-04~UC-12 |
| BR-03 | 이메일은 시스템 내 유일; 중복 시 HTTP 409 반환 | UC-01 |
| BR-04 | 회원가입 시 기본 카테고리(일반·업무·개인, is_default=true) 3개 자동 생성 | UC-01 |
| BR-05 | 기본 카테고리(is_default=true)는 삭제·수정 불가 | UC-07 |
| BR-06 | 할일에 이미 연결된 카테고리는 삭제 불가; 시도 시 HTTP 409 반환 | UC-07 |
| BR-07 | 할일 등록·수정 시 종료예정일은 서버 KST 기준 오늘 날짜 이상이어야 함 (당일 허용) | UC-08, UC-09 |
| BR-08 | 완료된 할일도 완료 취소(토글) 가능 | UC-11 |
| BR-09 | 사용자 탈퇴 시 status=withdrawn으로 변경(소프트 삭제); 탈퇴 계정으로 재가입 불가 | UC-05 |
| BR-10 | 비밀번호는 최소 8자, 영문+숫자 각 1자 이상 포함 | UC-01, UC-04 |

## 7. 유스케이스 목록

### UC-01 회원가입

| 항목 | 내용 |
|------|------|
| 액터 | 비인증 사용자 |
| 사전조건 | 사용자가 시스템에 로그인되어 있지 않다. |
| 기본 흐름 | 이메일, 비밀번호, 이름을 입력하고 제출한다. 시스템은 유효성 검사 후 계정을 생성하고 기본 카테고리 3개(일반·업무·개인)를 자동 생성한다. |
| 사후조건 | 신규 User 레코드(status=active)와 기본 Category 3개가 생성된다. |
| 예외 | `E01` 이메일 중복 → HTTP 409 "이미 사용 중인 이메일입니다." |
| | `E02` 비밀번호 정책 미충족 → HTTP 400 "비밀번호는 8자 이상, 영문과 숫자를 각 1자 이상 포함해야 합니다." |
| | `E03` 필수 항목 누락 → HTTP 400 "필수 입력값이 누락되었습니다." |
| 관련 BR | BR-03, BR-04, BR-10 |

| 항목 | 내용 |
|------|------|

| | `E03` 필수 항목 누락 → HTTP 400 "필수 입력값이 누락되었습니다." |

| 기본 흐름 | 자신의 할일 목록을 조회한다. 카테고리, 기간, 완료여부 필터를 단독 또는 복합으로 적용할 수 있다. |
| 사후조건 | 필터 조건에 부합하는 자신 소유의 할일 목록이 반환된다. 결과가 없으면 빈 목록을 반환한다. |
| 예외 | `E01` 유효하지 않은 날짜 형식 → HTTP 400 "날짜 형식이 올바르지 않습니다. (YYYY-MM-DD)" |
| | `E02` 시작일이 종료일보다 늦음 → HTTP 400 "시작일은 종료일보다 이전이어야 합니다." |
| 관련 BR | BR-01, BR-02 |

## 8. 할일 목록 필터 조건

| 필터 항목 | 파라미터 | 타입 | 설명 |
|-----------|----------|------|------|
| 카테고리 | category_id | UUID | 특정 카테고리에 속한 할일만 조회 |
| 기간 시작 | due_date_from | Date (YYYY-MM-DD) | 종료예정일이 해당 날짜 이상인 할일 조회 |
| 기간 종료 | due_date_to | Date (YYYY-MM-DD) | 종료예정일이 해당 날짜 이하인 할일 조회 |
| 완료 여부 | is_completed | Boolean | true: 완료된 할일 / false: 미완료 할일 / 생략: 전체 |

- 필터는 복합 적용 가능 (AND 조건으로 처리)
- 필터 미적용 시 사용자 소유의 전체 할일 반환
- 결과가 없을 경우 빈 배열([]) 반환 (HTTP 200)

## 10. 예외 처리 정책

### 10.1 HTTP 오류 응답 기준

| HTTP 코드 | 의미 | 적용 시나리오 |
|-----------|------|---------------|
| 400 Bad Request | 잘못된 요청 | 필수값 누락, 형식 오류, 비즈니스 규칙 위반 (날짜, 비밀번호 정책 등) |
| 401 Unauthorized | 인증 실패 | 미로그인 접근, 자격증명 불일치, 만료된 세션 |
| 404 Not Found | 리소스 없음 | 존재하지 않거나 타인 소유 리소스 접근 (정보 노출 방지) |
| 409 Conflict | 중복·충돌 | 이메일 중복, 카테고리 이름 중복, 연결된 카테고리 삭제 시도 |
| 500 Internal Server Error | 서버 오류 | 예상치 못한 서버 내부 오류 |

### 10.2 네트워크 오류 처리

- 감지 조건: API 응답 타임아웃 또는 네트워크 연결 불가
- UI 처리: 경고 메시지 표시 후 현재 입력 중이던 폼 데이터를 초기화
- 경고 메시지: "네트워크 오류가 발생했습니다. 잠시 후 다시 시도해 주세요."
- 데이터 보존 정책: 네트워크 오류 발생 시 입력 중인 할일 데이터는 초기화(저장하지 않음)

## 11. 용어 정의 (Glossary)

| 용어 | 정의 |
|------|------|
| 인증(Authentication) | 사용자가 이메일과 비밀번호로 신원을 증명하는 과정 |
| 인가(Authorization) | 인증된 사용자가 특정 리소스에 접근할 권한이 있는지 확인하는 과정 |
| 소프트 삭제(Soft Delete) | 데이터를 물리적으로 삭제하지 않고 상태값(status=withdrawn)을 변경하여 비활성화하는 방식 |
| 기본 카테고리 | 회원가입 시 자동 생성되는 카테고리(일반·업무·개인); 삭제·수정 불가 |
| 사용자 정의 카테고리 | 사용자가 직접 추가한 카테고리(is_default=false); 연결된 할일이 없으면 삭제 가능 |
| 완료 토글 | 할일의 is_completed 값을 true↔false로 전환하는 동작 |
| KST | Korea Standard Time (UTC+9); 날짜 기준이 되는 시간대 |
| bcrypt | 비밀번호 해시에 사용하는 단방향 암호화 알고리즘 |
| TLS | Transport Layer Security; 네트워크 통신 암호화 프로토콜 |
| JWT | JSON Web Token; 로그인 성공 후 서버가 발급하는 서명된 인증 토큰. 클라이언트 Zustand 메모리에만 보관하고 `Authorization: Bearer <token>` 헤더로 전달하며 24시간 후 만료된다 |
| UUID | Universally Unique Identifier; 시스템 전체에서 고유한 128비트 식별자 |
| due_date | 할일의 종료 예정일; KST 기준 오늘 날짜 이상이어야 함 |
