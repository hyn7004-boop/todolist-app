# 10. TodoListApp - ASCII 와이어프레임

## 개요

본 문서는 TodoListApp의 모든 화면에 대한 ASCII 와이어프레임을 제시합니다.
- 총 7개 주요 화면
- 공통 모달 및 알림 컴포넌트
- 모바일(320px~767px)과 데스크톱(1024px+) 뷰
- 빈 상태(Empty State) 및 에러 상태 포함

## 공통 컴포넌트

### 데스크톱 레이아웃

```
┌─────────────────────────────────────────────────────────────────┐

│  카테고리     │                                                  │

│  ✓ 업무      │    (각 페이지별 콘텐츠가 여기 표시)                │

│  네비게이션    │                                                  │

│               │                                                  │
│               │                                                  │
└──────────────┴──────────────────────────────────────────────────┘
```

**주요 특징:**
- 좌측 고정 Sidebar (약 250px)
- 상단 Header (고정, 높이 60px)
- 우측 콘텐츠 영역 (메인 콘텐츠)
- 1024px 이상 데스크톱 화면에서 표시

### 확인 모달 (삭제)

```
┌──────────────────────────────────┐
│                                  │
│         ⚠️ 삭제 확인              │
│                                  │
│  이 할일을 삭제하시겠습니까?       │
│  (복구할 수 없습니다)             │
│                                  │
│       [취소]      [삭제]          │
│                                  │
└──────────────────────────────────┘
```

**주요 특징:**
- 모달 중앙 정렬
- 경고 아이콘 표시
- 명확한 메시지
- 취소/삭제 버튼 (삭제 버튼은 위험 색상: 빨강)
- 백그라운드: 반투명 검은색 오버레이

┌──────────────────────────────────┐

│                                  │

│  • 동일 이메일 재가입 불가          │
│                                  │
│  ☐ 위 내용을 이해했으며,          │
│    탈퇴를 진행합니다              │
│                                  │
│       [취소]     [탈퇴]           │
│                                  │
└──────────────────────────────────┘
```

**주요 특징:**
- 체크박스로 명시적 동의 필수
- 위험 사항 리스트 표시
- 탈퇴 버튼은 체크박스 미체크 시 비활성화
- 위험 색상 강조

### Toast 알림

```
┌─────────────────────────────────┐
│ ℹ️  할일이 저장되었습니다          │
└─────────────────────────────────┘

┌─────────────────────────────────┐
│ ✓ 성공적으로 삭제되었습니다        │
└─────────────────────────────────┘

┌─────────────────────────────────┐
│ ⚠️ 네트워크 오류가 발생했습니다     │
│    나중에 다시 시도해주세요         │
└─────────────────────────────────┘

┌─────────────────────────────────┐
│ ✕ 카테고리 이름이 중복입니다       │
└─────────────────────────────────┘
```

**주요 특징:**
- 화면 우측 상단에 자동 위치
- 타입별 아이콘: 정보(ℹ️), 성공(✓), 경고(⚠️), 오류(✕)
- 3~4초 후 자동 사라짐
- 스택 형식으로 여러 개 표시 가능

## 인증 화면

### 로그인 화면 (/login)

#### 화면 설명
비인증 사용자가 계정으로 로그인하는 화면입니다.
- 이메일과 비밀번호를 입력받음
- 로그인 실패 시 오류 메시지 표시
- 회원가입 링크 제공

#### 주요 컴포넌트
- 이메일 입력 필드 (필수)
- 비밀번호 입력 필드 (필수)
- 로그인 버튼
- 회원가입 링크
- 오류 메시지 영역

### 회원가입 화면 (/signup)

#### 화면 설명
신규 사용자가 계정을 만드는 화면입니다.
- 이메일, 비밀번호, 이름을 입력받음
- 각 필드별로 인라인 유효성 검사 메시지 표시
- 성공 시 /login으로 이동

#### 주요 컴포넌트
- 이메일 입력 필드 (필수, 이메일 형식)
- 비밀번호 입력 필드 (필수, 8자+영문+숫자)
- 이름 입력 필드 (필수, 최대 50자)
- 가입하기 버튼
- 로그인 링크
- 각 필드별 인라인 에러/도움말 텍스트

#### 데스크톱 뷰

```
┌────────────────────────────────────────────┐

## 메인 기능 화면

### 할일 목록 (메인) (/todos)

#### 화면 설명
사용자의 모든 할일을 조회하고 관리하는 메인 화면입니다.
- 필터 패널로 카테고리, 날짜, 완료 여부로 필터링
- 할일 목록 표시 (완료 여부에 따라 스타일 변경)
- 새 할일 추가 버튼
- 각 항목에서 수정/삭제 가능

#### 주요 컴포넌트
- 필터 패널 (카테고리, 기간, 완료 여부)
- 필터 적용/초기화 버튼
- 새 할일 추가 버튼
- 할일 목록 (항목별: 완료 체크박스, 제목, 카테고리 배지, 마감일, 수정/삭제 버튼)
- 빈 상태 메시지

## 색상 및 스타일 가이드

### 버튼 타입

| 타입 | 배경색 | 텍스트색 | 용도 |
|------|--------|---------|------|
| Primary | 파랑 | 흰색 | 저장, 로그인, 가입 |
| Secondary | 회색 | 검은색 | 취소, 초기화 |
| Danger | 빨강 | 흰색 | 삭제, 탈퇴 |
| Ghost | 투명 | 파랑 | 링크 형식의 버튼 |

### 상태 표시

| 상태 | 아이콘 | 색상 |
|------|--------|------|
| 성공 | ✓ | 초록색 |
| 오류 | ✕ | 빨강색 |
| 경고 | ⚠️ | 주황색 |
| 정보 | ℹ️ | 파랑색 |

### 텍스트 상태

## 접근성 고려사항

1. **폼 라벨:** 모든 입력 필드는 명확한 라벨 보유
2. **필수 표시:** 별(*) 기호로 필수 필드 표시
3. **에러 메시지:** 시각적 표시 + 텍스트 메시지 (색상 약자들을 위해)
4. **키보드 네비게이션:** Tab 키로 모든 상호작용 가능
5. **포커스 표시:** 명확한 포커스 테두리 표시
6. **대비:** WCAG AA 이상의 텍스트 대비
7. **모달:** 포커스 트랩 구현 (모달 내에서만 Tab 이동)

---

## 애니메이션 및 전환

1. **모달 진입/퇴출:** 부드러운 페이드인/아웃 (200ms)
2. **드로어 애니메이션:** 슬라이드인/아웃 (300ms)
3. **Toast 알림:** 슬라이드인 (200ms) → 자동 퇴출 (3초 후 200ms 슬라이드아웃)
4. **버튼 호버:** 배경색 변화 + 마우스 커서 포인터
5. **로딩 상태:** 버튼 비활성화 + 로딩 스피너 (선택)

---

## 반응형 고려사항

### 모바일 최적화

1. **터치 타겟:** 최소 44px × 44px
2. **입력 필드:** 모바일 키보드 대응
   - 이메일: `type="email"` (이메일 키보드)
   - 비밀번호: `type="password"` (마스킹)
   - 날짜: `type="date"` (날짜 픽커)
3. **간격:** 모바일에서 충분한 패딩/마진 확보
4. **폰트 크기:** 최소 16px (자동 줌 방지)

### 태블릿 최적화

1. **2컬럼 레이아웃:** 필터와 목록을 나란히 배치 (선택)
2. **사이드바:** 축소 또는 아래로 이동
3. **터치 친화적:** 모바일과 데스크톱의 중간 형태
